# 会展公司 CRM 1.0 方案（普通客户 / 主场客户）

## 1.0 目标与边界
- 目标：建立可闭环的商机管理体系，覆盖“商机收集 → 分配跟进 → 过程管理 → 转化移交 → 复盘沉淀”。
- 范围：仅做商机管理与转化流程（不含客户主数据管理、Salesforce 数据推送、AI 能力与宣传获客）。
- 商机类型：普通商机（搭建业务）与主场商机（展会主场业务）。
- 组织权限：集团公司拥有全量数据访问与配置权限；子公司仅可访问本公司数据。

## 核心模块与页面
1) 商机池（Opportunity Pool）
   - 多渠道导入、清洗、去重、标签化、分配。
2) 商机管理
   - 普通商机列表 / 详情
   - 主场商机列表 / 详情（含主办方关系、展会信息）
3) 跟进与任务
   - 跟进记录、待办清单、提醒与超时
4) 数据看板
   - 有效率、跟进及时率、阶段转化率、来源分布、业务类型占比
5) 权限与组织隔离
   - 集团/子公司数据隔离、个人可见范围、管理员全局配置

## 商机分类与定义
### 普通商机（搭建业务）
- 来源：网上公开数据、Salesforce 流失客户、Onsite/个人关系客户、历史中标客户等。
- 目标：高效触达与重复利用，提升转化率。

### 主场商机（展会主场业务）
- 结构：主办方 → 展会 → 关键负责人
- 分类：外资、国企、政企合办、政府类（如 CRE）、纯商业类主办方
- 目标：主办方关系沉淀与新展机会发现。

## 流程与阶段（1.0）
1) 潜客发掘
   - 多渠道导入、去重、有效性标注（电话/邮箱/微信）。
2) 跟进分配
   - 明确负责人、触达方式（邮件/电话/微信）。
3) 管理监控
   - 跟进有效率、未跟进清单、超时提醒。
4) 成交转化
   - 形成投标意向的商机进入“转化待移交”状态（仅标记）。
5) 分析复盘
   - 复购可能性判断、标签更新、主办方关系图谱完善。

## 数据模型（核心实体）
### 商机 Opportunity
必填字段：
- 商机名称、来源、商机类型（普通/主场）、行业、城市、状态
可选字段：
- 展会名称、主办方、负责人、联系方式（电话/邮箱/微信）、备注

### 客户 Account（只读/引用）
说明：客户主数据在现有 CRM 内维护，本系统仅做引用与关联。
必填字段（引用）：
- 客户名称、客户类型、行业、城市、状态、负责人（Owner）、BU

### 联系人 Contact（只读/引用）
必填字段：
- 姓名、职位、联系方式、所属客户
可选字段：
- 影响力等级、关系温度、备注

### 跟进记录 Activity
- 触达方式、时间、结果、下一步计划、附件/资料链接

### 标签 Tag
- 城市、行业、跟进状态、业务类型（搭建/主场）、展会品类

### 主场扩展对象（1.0 轻量化）
- 主办方 Organizer（名称、类型、关键负责人、关系阶段）
- 展会 Exhibition（名称、城市、周期、规模、负责人）

## 状态与转化规则
- 商机状态：新建 → 已分配 → 跟进中 → 有效 → 无效
- 商机阶段：冷淡 → 意向 → 需求明确 → 投标准备 → 转化待移交
- 转化触发：达到“投标准备”，进入“转化待移交”状态并锁定关键信息。

## 数据清洗与去重
- 去重规则：商机名称 + 城市（主键）；手机号/邮箱作为强唯一。
- 无效标记：空号/停机/拒绝/长期无响应（>3 次）。
- 版本沉淀：保留原始来源字段，标记最后更新时间。

## 权限与组织隔离
- 角色：集团管理员、子公司管理员、销售、市场支持。
- 权限原则：
  - 集团管理员：全量可见与全局配置。
  - 子公司管理员：本公司全量可见与配置。
  - 销售：仅本人及共享商机可见。

## 看板指标（1.0）
- 商机有效率 = 有效商机 / 总商机
- 跟进及时率 = 24h 内有跟进 / 已分配商机
- 阶段转化率（分商机类型）
- 来源分布（5 类来源占比）
- 主场商机：主办方类型占比、关键负责人覆盖率

## 录入与导入规范
### 导入字段（最低要求）
- 商机名称、商机类型、来源、城市、行业、负责人、联系方式
### 录入校验
- 联系方式至少一项；客户名称不可为空。

## 与 Salesforce 的下阶段衔接（预留）
- 1.0 只做字段对齐与状态标记，不做自动同步。
- 预留字段：Salesforce_ID、同步状态、同步时间。

## 1.0 原型结构（页面树）
- 首页看板
- 线索池
- 客户列表（普通/主场）
- 客户详情（含联系人、跟进记录、附件）
- 主场图谱（主办方-展会-负责人关系视图）
- 导入中心
- 权限与字段配置
